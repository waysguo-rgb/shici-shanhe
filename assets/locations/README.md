# 诗词山河 · 地点 & 诗词数据集

这个目录包含诗词山河地图里所有 **60 个地点** 和 **165 首诗** 的结构化数据，
以及每个地点对应的**代表建筑/风景**，方便你后续去找配图或生成 3D 模型。

## 文件说明

| 文件 | 格式 | 内容 |
|---|---|---|
| `locations.json` | JSON | 完整数据，嵌套结构（地点 → 诗词列表） |
| `locations_summary.csv` | CSV (UTF-8 BOM) | 每个地点一行：名称 / 经纬度 / 诗词数 / 代表景观 |
| `poems.csv` | CSV (UTF-8 BOM) | 每首诗一行：地点 / 经纬度 / 标题 / 作者 / 朝代 / 全文 / 该地点代表景观 |

CSV 带 UTF-8 BOM，Excel 直接打开不乱码。

## 数据结构（JSON）

```json
{
  "name": "长安",
  "lon": 108.94,
  "lat": 34.26,
  "icon": "🏯",
  "representative_landmarks": [
    "大明宫含元殿",
    "未央宫遗址",
    "大雁塔",
    "小雁塔",
    "曲江池",
    "乐游原",
    "灞桥",
    "城墙角楼"
  ],
  "poem_count": 6,
  "poems": [
    {
      "title": "春望",
      "author": "杜甫",
      "dynasty": "唐",
      "lines": ["国破山河在", "城春草木深", "..."],
      "hl": true,
      "excerpt": false
    }
  ]
}
```

字段：
- `hl` — 是否作为该地点的"代表作"高亮展示
- `excerpt` — 是否为节选（长诗如 `长恨歌`、`蜀道难`）
- `representative_landmarks` — 按文化意义排列的代表景观/建筑，**用于生成 3D 模型**

## 覆盖范围

- **60** 个地点，从**长安**到**邯郸**
- **165** 首诗词
- 每地 2–6 首，**每地 3–8 个代表景观**
- 朝代跨度：**汉 → 清**

## 用途

### A. 找图 → 人工建模

1. 打开 `locations_summary.csv`
2. 按 `representative_landmarks` 一列去搜图（建议 Google Images、百度图片、故宫博物院、国家博物馆的数字藏品）
3. 给每个景观准备 4–8 张不同角度的参考图
4. 用建模软件（Blender、3ds Max、ZBrush）或 AI 生模（Luma Genie / Meshy / Tripo）做出 glTF / OBJ 格式的 3D 模型

### B. 直接用 AI 文生模

很多文生 3D 模型（text-to-3D）工具支持中文 prompt：

- **Meshy.ai** — 输入"大雁塔，唐代砖塔，七层，方形"
- **Tripo3D** — 类似，生成 GLB- **Luma Genie** — 图生 3D 更精细
- **Hunyuan3D (腾讯混元)** — 中文支持好，对中国古建筑效果较好

输出的文件格式建议：**glTF / GLB**（Three.js 直接支持）

### C. 导入到地图

准备好 3D 模型后：

1. 把 `.glb` 文件放到 `assets/models/<地点名>/<景观名>.glb`，例如：
   ```
   assets/models/长安/大雁塔.glb
   assets/models/长安/大明宫含元殿.glb
   ```

2. 我可以帮你写 Three.js 加载代码（使用 `GLTFLoader`），在诗词山河.html 里：
   - 读取 JSON 数据
   - 对每个有模型的景观，创建一个 `Group` 放在对应地点的经纬度位置
   - 拉近时显示，拉远时隐藏（或直接接入 LOD 系统）
